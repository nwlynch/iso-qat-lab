# Agent Interaction Protocol Contract

This document defines the mandatory structure for all agent outputs when reporting results back to the Orchestrator (e.g., the main shell script or session status checks). This ensures seamless, structured communication between specialized agents.

## Output Format: Standardized JSON

All agents **must** return their final result in a standardized JSON format.

### Mandatory Fields:

1.  **`status` (String):** The final outcome of the task. Must be one of: `SUCCESS`, `FAILURE`, or `WARNING`.
2.  **`result_data` (Object/String):** The core output or data generated by the agent. This should contain the primary artifact requested by the workflow step.
3.  **`metadata` (Object):** Contextual information about the execution.
    *   `agent_name` (String): The name of the agent that performed the task.
    *   `task_name` (String): The specific task executed (e.g., "Parse requirements").
    *   `iso_clause` (String): The specific ISO/IEEE clause addressed (e.g., "12207.3.1").
    *   `model_used` (String): The LLM model used for this execution.
    *   `time_taken_ms` (Integer): The time taken for the agent to complete the task in milliseconds.
4.  **`error` (String, Optional):** Detailed error message if `status` is `FAILURE`.

### Example Structure:

```json
{
  "status": "SUCCESS",
  "result_data": {
    "requirements_parsed": [
      {"id": "REQ-001", "description": "System must handle X input.", "criteria": ["Y", "Z"]}
    ]
  },
  "metadata": {
    "agent_name": "acq-requirements-parser",
    "task_name": "Parse requirements",
    "iso_clause": "12207.3.1",
    "model_used": "qwen3.5:9b",
    "time_taken_ms": 1500
  }
}
```

## Orchestration Note:

The main workflow runner (`run-full-lifecycle.sh`) will be responsible for capturing the output from the spawned sessions, parsing the JSON response, and feeding that data into the next step in the chain (e.g., feeding the structured requirements from `acq-requirements-parser` into `pm-project-planning`).
